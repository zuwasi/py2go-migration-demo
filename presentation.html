The Problem

AI Coding Agents Fail at
Real-World Migrations

Winder.ai migrated a production Python codebase to Go using Claude Code.
The code compiled. Tests passed. The system didn't work.

👻
Phantom Features — AI rebuilt a deprecated database table from dead code references, causing zero search results
💥
Context Window Blowup — Cross-cutting tasks missed half the endpoints; AI paginated in-memory instead of at the DB
🏚️
Architectural Drift — Everything landed in internal/ — no public API was ever created
🧟
Dead Code Accumulation — Orphaned functions went unnoticed for days across refactors
🔇
No Integration Tests — Unit tests passed but search returned completely wrong results end-to-end

Root Cause

A Bash Loop Is Not a
Migration Strategy

# The Claude Code approach: a blind loop
while true; do
  claude --print "Continue migration..."
  git add -A && git commit
  sleep 5
done

❌ No call-graph analysis

Dead symbols are migrated, creating phantom features that break the system silently.

❌ Single context window

Large cross-cutting tasks exceed the window. The AI loses track and misses work.

❌ No scoped rules

One monolithic CLAUDE.md — no way to enforce different standards per directory.

❌ No verification loop

The server is never started. Dead code is never swept. Problems compound silently.

The Amp Solution

Structured, Verified, Automated

Amp provides native tools that prevent every failure from the article.

✅ Pre-Migration Dead Code Filter

Call-graph analysis BEFORE migration. Dead symbols are skipped and logged — no phantom features.

✅ Parallel Sub-Agents

Cross-cutting tasks spawn one sub-agent per endpoint. Each gets its own context window — nothing missed.

✅ Directory-Scoped AGENTS.md

Root rules for translation tables. Go-specific rules scoped to go-target/. Right context, right files.

✅ Dead Code Sweep Every Task

Automated after EVERY task — not a manual afterthought. Orphaned code is caught immediately.

✅ Smoke Tests via tmux

Server starts, endpoints are curled, results compared to Python baseline at every phase.

✅ Ralph Task Engine

Dependency-aware task list. Ralph picks the next ready task, executes, verifies, commits. Fully autonomous.
